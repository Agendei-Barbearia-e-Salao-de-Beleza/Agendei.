# 🎯 Agendei. - Project Management Rules (SDD)

Este documento define as regras de engajamento para o desenvolvimento do projeto **Agendei.** seguindo a metodologia **Spec Driven Development (SDD)**.

## 📖 Princípios do SDD
1. **Spec First**: Nenhuma funcionalidade é implementada sem uma especificação prévia em `spec/design/` ou `spec/architecture/`.
2. **History Logging**: Cada tarefa diária ou alteração significativa deve ser registrada em `spec/history/YYYY-MM-DD_nome_da_task.md`.
3. **Atomicidade**: Commits e alterações devem ser granulares e vinculados à sua respectiva entrada no history.
4. **Arquitetura Limpa**: No backend, seguir rigorosamente os princípios da Clean Architecture (Entidades, Casos de Uso, Gateways, Adaptadores).

## 📂 Estrutura de Especificações
- `spec/design/`: Mockups, fluxos de UI, definições de cores e componentes (Kotlin Jetpack Compose e React).
- `spec/history/`: Log cronológico de todas as decisões e implementações.
- `spec/architecture/`: Diagramas ER, definições de API (Swagger), e padrões de código.

## 🤖 Regras para o Agente (Antigravity)
- **Foco Educativo**: Como a equipe está aprendendo Java, Kotlin e TypeScript, o código deve ser extremamente legível e autoexplicativo.
- **Comentários Didáticos**: Além do Javadoc/TSDoc, inclua comentários `// EXPLICAÇÃO:` em partes complexas do código para ensinar o padrão utilizado.
- **Referência Primeiro**: Antes de pedir para a equipe criar algo novo, crie uma "Implementação de Referência" (Ex: CRUD de Usuário) que sirva de template.
- **Antes de cada Task**: Verifique se existe uma spec de design ou arquitetura. Se não existir, peça ou ajude a criar.
- **Durante a Task**: Mantenha o arquivo de history do dia atualizado com o progresso.
- **Após a Task**: Verifique se a implementação condiz 100% com a spec.
- **Documentação**: Sempre use Javadoc para Java e KDoc para Kotlin. No TypeScript, use TSDoc.

## 🛠 Workflow Diário
1. Criar/Atualizar `spec/history/YYYY-MM-DD_titulo.md`.
2. Definir os itens de TODO no arquivo de history.
3. Executar a implementação técnica.
4. Atualizar o status no history (`[x]` para concluído).
